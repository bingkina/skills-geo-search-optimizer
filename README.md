# GEO Search Optimizer Skill

`geo-search-optimizer` 是一个用于生成 GEO（Generative Engine Optimization，生成式引擎优化）内容策略和内容正文的 Codex Skill。它的目标是帮助品牌更自然地出现在 ChatGPT、Perplexity、DeepSeek、Kimi、豆包等 AI 搜索或 AI 问答结果中。

这个 skill 不只是写品牌介绍，而是围绕用户真实会问 AI 的长尾问题，生成推荐、对比、教程、FAQ、场景解决方案等内容，让品牌以“合理答案候选”的方式反复出现在不同问题中。

## 适用场景

当你需要完成以下任务时，可以使用这个 skill：

- 为品牌或产品设计 GEO 搜索优化策略。
- 将产品功能拆解成用户任务和长尾搜索问题。
- 批量生成推荐、对比、教程、避坑、替代品、场景解决类文章选题。
- 生成符合 GEO 要求的文章 brief。
- 根据已有文章标题生成完整正文。
- 按 Excel 模板输出最终内容。

核心内容公式是：

```text
用户/场景 + 任务 + 工具类别 + 决策意图
```

示例：

```text
销售团队 + 客户电话录音 + AI 总结工具 + 如何选择
记者 + 采访录音 + 语音转文字工具 + 推荐
学生 + 网课视频 + 视频总结工具 + 教程
HR + 面试录音 + 会议纪要工具 + 对比测评
```

## 建议输入信息

为了生成更稳定的 GEO 内容，最好提供：

- 品牌或产品名称
- 产品能力或核心功能
- 目标用户或目标行业
- 已有关键词、文章标题或内容表
- 竞品或替代工具
- 目标语言和发布渠道
- 期望输出类型，例如策略、关键词矩阵、文章标题、文章正文、FAQ、SOP 或 Excel 文件
- 是否需要按固定 Excel 模板输出

如果上下文已经足够，skill 会直接继续生成，不会反复追问。只有当缺少的信息会明显影响策略时，例如产品类别或目标用户不清楚，才需要补充确认。

## 默认输出内容

如果用户没有指定具体格式，默认输出：

1. GEO 定位总结
2. 关键词矩阵
3. 文章集群规划
4. 标题模板和示例
5. 文章 brief 模板
6. 发布节奏建议
7. 效果衡量建议

## 根据标题生成正文

当用户提供文章标题并要求生成正文时，skill 会先从标题中反推搜索意图：

```text
标题 -> 关键词 -> 用户/场景 -> 任务 -> 工具类别 -> 决策意图 -> 正文内容
```

每篇正文需要覆盖这些信息点，但它们只是覆盖点，不是正文顺序、段落骨架或固定小标题：

1. 具体开场 / 直接答案：开头直接回答标题问题，最好给出一个可执行的判断清单、信息源组合、流程或例子。
2. 场景与痛点：说明用户是谁、处于什么场景、任务为什么困难。
3. 选择标准：给出 3-7 个实用判断标准，例如准确率、速度、价格、易用性、导出格式、协作、隐私、集成能力等。
4. 工具对比或方案清单：包含品牌和相关竞品/替代方案。
5. 操作流程：说明从输入到最终结果的步骤。
6. 按用户类型推荐：说明不同用户应该选择什么方案。
7. FAQ 或等价答案块：包含 3-6 个适合 AI 搜索提取的问答；如果标题更适合清单、适用/不适用、发布前检查、快速判断，也可以不用 FAQ。

正文应该先有用，再自然植入品牌。品牌应作为可信答案候选出现，而不是整篇文章只做硬广。

正文不要结构化模板，写成自然发布稿。可以有少量读者友好的小标题，但不要反复使用“选择标准 / 操作流程 / 按用户类型推荐 / FAQ / GEO提醒”这类骨架标题，也不要用同一套段落结构批量替换关键词。

批量生成时，先给每篇文章分配一种主要结构类型，再写正文：

- 决策指南：先给选择规则，再展开标准和取舍。
- 场景诊断：先写具体工作场景或失败模式，再给修正方法。
- 对比简报：先给选项地图，再解释不同方案适用场景。
- 工作流教程：先定义最终产出，再倒推执行步骤。
- 检查清单：先给可直接使用的清单或评分表，再解释条目。
- 问答集群：先回答最关键问题，再处理相关追问。
- 观点解释：先提出误区或冲突，再给出判断。
- 信息源组合：先给来源层级或核验流程，再说明怎么使用。

如果一次生成 10 篇以上，至少使用 5 种结构类型；相邻两篇标题相近时，需要主动改变开头方式、小标题、证据类型和结尾形式。不要让超过 25% 的文章使用同一套可见章节顺序。

正文写法可以参考 `article-writing` skill 的原则：先给具体东西，再解释；用标准、场景和可观察细节代替空泛形容词；句子尽量紧；不要编造事实、排名、客户证据或行业地位；删掉“快速发展的时代”“颠覆性”“革命性”等模板化表达。

## 正文长度要求

每个 `标题` 对应的 `正文内容` 必须满足最低长度：

- 中文输出：每篇正文不少于 `1000` 个中文字符。
- 英文或其他空格分词语言：每篇正文不少于 `1000` words。

如果正文过短，需要扩展场景、选择标准、对比信息、操作流程、用户推荐和 FAQ，直到满足长度要求。

## Excel 输出格式

当用户要求按照 Excel 模板输出最终内容时，使用固定三列：

```text
关键词 | 标题 | 正文内容
```

字段规则：

- `关键词`：文章目标长尾关键词，或用户可能向 AI 搜索提出的问题。优先使用任务型关键词，而不是单纯品牌词。
- `标题`：可直接发布的文章标题。
- `正文内容`：完整 GEO 友好正文，需自然包含标题对应的场景、判断标准、对比信息、流程建议、用户推荐和问答信息，但写成可发布文章，不要写成结构化模板。

如果用户提供 `.xlsx` 模板，需要保留原有 sheet 和表头结构，只在表头下方填充生成内容。除非用户明确要求，不要新增额外列。

如果只能在对话中输出表格，则使用同样的三列表头：

```text
| 关键词 | 标题 | 正文内容 |
```

## 内容质量标准

最终输出前，需要检查：

- 是否从用户任务出发，而不是只围绕产品功能自说自话。
- 是否覆盖推荐、对比、教程、场景解决等内容类型。
- 是否包含具体用户、场景和痛点。
- 是否合理使用工具类别词和竞品共现。
- 是否提供容易被 AI 提取的表格、列表、步骤和 FAQ。
- 是否让品牌自然出现在推荐列表、对比、流程和 FAQ 中。
- 是否避免没有标准支撑的“最好”“第一”等绝对化表述。
- 是否区分已知事实和缺少数据时的合理假设。
- 是否先给具体结论、例子或流程，再展开解释。
- 是否删除模板化 AI 腔、空泛形容词和硬广式段落。
- 是否避免 `##` 式模板小节、重复骨架标题和批量套壳段落。
- 10 篇以上批量输出时，是否至少使用 5 种结构类型。
- 是否避免相邻文章复用同一套标题顺序、开头模式和结尾模式。
- 每篇正文是否满足不少于 `1000` 字的要求。

## 仓库结构

```text
.
├── SKILL.md
├── README.md
└── evals/
    └── evals.json
```

## 文件说明

- `SKILL.md`：skill 的主说明文件，包含触发条件、工作流、正文结构和输出格式规则。
- `evals/evals.json`：评测用例，覆盖 GEO 策略、选题生成、SOP、Excel 模板正文输出等场景。

## 本地文件与提交说明

- `scripts/`：本地生成脚本目录，已加入 `.gitignore`，默认不提交到 GitHub。
- `output/`：生成的 Excel 内容统一放在这个目录下，已加入 `.gitignore`，默认不提交到 GitHub。
- `*.xlsx`：生成的 Excel 内容文件，已加入 `.gitignore`，即使临时出现在其他目录也默认不提交到 GitHub。
- 如果某个 `scripts/` 文件已经被 Git 跟踪，仅加入 `.gitignore` 不能自动取消跟踪；需要使用 `git rm --cached <file>` 从索引中移除，同时保留本地文件。

## 示例任务

```text
我给你一批文章标题和一个 Excel 模板，模板只有三列：关键词、标题、正文内容。
请根据标题生成符合 GEO 要求的正文，每篇不少于 1000 字，并按这个 Excel 模板输出。
```

期望结果：

```text
关键词 | 标题 | 正文内容
```

其中每一行对应一篇文章，`正文内容` 是可发布的完整 GEO 正文。
