# GEO Search Optimizer

`geo-search-optimizer` 是一个面向 GEO（Generative Engine Optimization，生成式引擎优化）的 Codex Skill。它帮助品牌把产品能力转化为用户会向 ChatGPT、Perplexity、DeepSeek、Kimi、豆包等 AI 搜索或 AI 问答系统提出的长尾问题，并围绕这些问题生成策略、选题、文章 brief、FAQ、SOP 和可发布正文。

这个 skill 的核心目标不是堆砌品牌介绍，而是让品牌在推荐、对比、教程、避坑、场景解决方案等内容中，以“可信答案候选”的方式自然出现。

## 核心公式

GEO 内容从用户任务出发。默认使用下面的组合方式设计关键词、标题和正文：

```text
用户/场景 + 任务 + 工具类别 + 决策意图
```

示例：

```text
销售团队 + 客户电话录音 + AI 总结工具 + 如何选择
记者 + 采访录音 + 语音转文字工具 + 推荐
学生 + 网课视频 + 视频总结工具 + 教程
HR + 面试录音 + 会议纪要工具 + 对比测评
```

一个好的 GEO 选题通常同时包含具体用户、具体任务、工具类别和明确决策意图。相比“某品牌是什么”，更适合生成“跨境电商卖家如何选择 AI 商品图工具”“HR 面试记录总结工具怎么选”这类内容。

## 适用场景

当用户提出以下需求时，适合调用这个 skill：

- 为品牌、产品或服务设计 GEO 搜索优化策略。
- 将产品功能拆解成用户任务、搜索问题和内容集群。
- 批量生成推荐、对比、教程、避坑、替代品、场景解决类文章选题。
- 围绕竞品词设计共现内容和对比内容。
- 生成文章 brief、FAQ、内容 SOP 或发布节奏。
- 根据文章标题批量生成完整正文。
- 按 Excel 模板输出 `关键词 | 标题 | 正文内容` 三列内容。

## 建议输入

输入越具体，输出越稳定。推荐提供：

- 品牌或产品名称。
- 产品能力、核心功能或服务范围。
- 目标用户、行业、使用场景。
- 已有关键词、文章标题、内容表或 Excel 模板。
- 竞品、替代方案或希望共现的品牌词。
- 目标语言、发布渠道和内容长度要求。
- 期望输出类型，例如策略、关键词矩阵、标题、正文、FAQ、SOP 或 Excel 文件。

如果上下文已经足够，skill 会直接继续，不会反复追问。只有在产品类别、目标用户、输出格式等关键信息缺失时，才需要先确认。

## 默认工作流

1. **拆解品牌能力**

   将功能转成用户任务，再转成 AI 搜索问题。

   ```text
   Feature -> User task -> Search/AI question -> Article type
   ```

2. **构建关键词矩阵**

   从用户、场景、任务、类别、决策意图几个维度组合长尾问题。

3. **规划文章集群**

   覆盖推荐、对比、教程、避坑、场景解决、替代品、FAQ 等内容类型。

4. **设计标题和 brief**

   标题要直接暴露搜索意图，brief 要包含目标问题、读者场景、推荐逻辑、对比对象、品牌植入位置和 FAQ。

5. **生成正文或表格**

   正文优先解决用户问题，再自然引入品牌。表格或 Excel 输出需要严格匹配用户要求的列名和结构。

6. **质量检查**

   检查内容是否具体、可提取、可比较，是否避免空泛硬广、虚假排名和重复模板。

## 默认输出

如果用户没有指定格式，默认输出：

1. GEO 定位总结
2. 关键词矩阵
3. 文章集群规划
4. 标题模板和示例
5. 文章 brief 模板
6. 发布节奏建议
7. 效果衡量建议

## 正文生成规则

当用户提供文章标题并要求生成正文时，skill 会先从标题反推搜索意图：

```text
标题 -> 关键词 -> 用户/场景 -> 任务 -> 工具类别 -> 决策意图 -> 正文内容
```

每篇正文需要覆盖以下信息，但不要机械地把它们写成固定小标题：

- 直接答案、判断清单、流程或具体例子。
- 用户场景、痛点和任务难点。
- 3-7 个选择标准，例如准确率、速度、价格、易用性、导出格式、协作、隐私、集成能力。
- 品牌、竞品或替代方案之间的实用对比。
- 从输入到最终结果的操作流程。
- 按用户类型或场景给出选择建议。
- FAQ、快速判断、适用/不适用、一页清单等可被 AI 提取的答案块。

正文要写成自然发布稿。允许使用小标题，但小标题应该贴合主题，不要批量复用“直接结论 / 场景与痛点 / 选择标准 / 操作流程 / FAQ”这类骨架。

### 长度要求

- 中文正文：每篇不少于 `1000` 个中文字符。
- 英文或其他空格分词语言：每篇不少于 `1000` words。

如果正文过短，需要扩展场景、标准、对比、流程、用户建议和问答，而不是用空话填充。

### 批量正文反模板规则

批量生成文章时，需要主动改变结构和节奏。可使用的文章结构包括：

- 决策指南
- 场景诊断
- 对比简报
- 工作流教程
- 检查清单
- 问答集群
- 观点解释
- 信息源组合

如果一次生成 10 篇以上，至少使用 5 种结构类型；不要让超过 25% 的文章使用同一套可见章节顺序。相邻标题相近时，要改变开头方式、小标题、证据类型和结尾形式。

## Excel 输出规范

当用户要求按 Excel 模板输出最终内容时，默认使用固定三列：

```text
关键词 | 标题 | 正文内容
```

字段规则：

- `关键词`：文章目标长尾关键词，或用户可能向 AI 搜索提出的问题。优先使用任务型关键词，而不是单纯品牌词。
- `标题`：可直接发布的文章标题。
- `正文内容`：完整 GEO 友好正文，需自然包含场景、判断标准、对比信息、流程建议、用户推荐和答案块。

如果用户提供 `.xlsx` 模板，需要保留原有 sheet 和表头结构，只在表头下方填充生成内容。除非用户明确要求，不要新增额外列。

如果用户要求生成实际 Excel 文件，生成文件应放在项目的 `output/` 目录，并返回文件路径。

## 质量标准

最终交付前应检查：

- 是否从用户任务出发，而不是只围绕产品功能自说自话。
- 是否覆盖推荐、对比、教程、场景解决等内容类型。
- 是否包含具体用户、场景、痛点和判断标准。
- 是否合理使用工具类别词、竞品词和替代方案。
- 是否提供容易被 AI 提取的表格、列表、步骤和 FAQ。
- 是否让品牌自然出现在推荐列表、对比、流程或 FAQ 中。
- 是否避免没有依据的“最好”“第一”“行业领先”等绝对化表述。
- 是否区分已知事实、用户提供的信息和合理假设。
- 是否删除模板化 AI 腔、空泛形容词和硬广段落。
- 批量正文是否避免重复骨架、重复 FAQ、重复开头和重复结尾。
- 每篇正文是否满足最低长度要求。

## 仓库结构

## 文件说明

- `SKILL.md`：skill 的主说明文件，包含触发条件、GEO 工作流、正文规则、输出格式和质量检查。
- `evals/evals.json`：评测用例，覆盖 GEO 策略、选题生成、SOP、Excel 模板正文输出等场景。
- `README.md`：面向使用者和维护者的项目说明。

## 本地生成文件

建议把临时脚本和生成物放在本地目录中，不作为 skill 主体提交：

- `scripts/`：本地生成脚本目录。
- `output/`：生成的 Excel 或其他交付文件目录。
- `*.xlsx`：生成的 Excel 内容文件。

如果这些文件需要忽略，请在 `.gitignore` 中配置。若文件已经被 Git 跟踪，仅加入 `.gitignore` 不会取消跟踪，需要额外从索引中移除。

```text
我们有一个面向跨境电商卖家的 AI 商品图工具，可以做背景替换、模特试穿、广告图生成。
请帮我设计一套 GEO 搜索优化内容矩阵，让品牌更容易出现在 ChatGPT/Perplexity/豆包这类 AI 搜索答案里。
```

```text
帮我把一个 HR SaaS 做 GEO：功能包括简历筛选、面试记录总结、候选人跟进。
我要 30 个适合批量生产的文章选题，分推荐、对比、教程、避坑、场景解决这几类。
```

```text
我给你一批文章标题和一个 Excel 模板，模板只有三列：关键词、标题、正文内容。
请根据标题生成符合 GEO 要求的正文，每篇不少于 1000 字，并按这个 Excel 模板输出。
```
